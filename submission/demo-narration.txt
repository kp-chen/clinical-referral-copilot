Referral triage is high stakes. This copilot offers a pattern: AI structures; clinicians decide.

Using only synthetic gastroenterology referrals, a clinician reviews the narrative and asks for a structured urgency recommendation. The analysis runs securely on the server, so the API key never reaches the browser.

GPT five point six uses the OpenAI Responses API with strict Structured Outputs and no storage. It returns urgency, confidence, facts, missing information, and reasons. Selecting a reason highlights the exact source phrase and its synthetic rule.

The model cannot take clinical action. A clinician must approve, edit, or reject every recommendation, while the audit records that decision without storing the underlying referral narrative.

The evaluation view separates evidence from persuasion. This tiny synthetic gold set is illustrative: four labels agree, while one disagreement remains visible.

Identifiers, phone numbers, email addresses, and high-confidence prompt-injection patterns are blocked before any model call. Evidence appears only when every quoted phrase is verified for exact matching against the supplied narrative.

Codex helped turn the clinical concept into a clean-room Next.js build, shape the clinician-in-the-loop boundary, harden identifier and prompt-injection guards, and verify the API and browser workflow.

This is not a medical device and makes no clinical-performance claim. It is a Build Week prototype demonstrating transparent, clinician-in-the-loop reasoning: synthetic data, inspectable evidence, deliberately bounded agency, and final human authority.
